# Ea_max IT Services (RSU) — Website

A complete, production-ready website for **Ea_max IT Services**, an educational
consultancy run by Mr. Ernest Anabs at the Rivers State University Shopping
Complex, Port Harcourt.

Built with **React + TypeScript + Tailwind CSS + React Router**. There is no
backend — every order and enquiry is routed to WhatsApp with a pre-filled
message.

---

## Tech stack

- React 18 + TypeScript
- Vite (build tool / dev server)
- Tailwind CSS 3
- React Router 6 (multi-page)
- Deploy target: Netlify (free tier)

## Pages

| Route        | Page      | Highlights                                                        |
| ------------ | --------- | ---------------------------------------------------------------- |
| `/`          | Home      | Hero, scrolling deadline ticker, services preview, why-us, testimonials, CTA |
| `/services`  | Services  | Full catalogue with category filters and WhatsApp order buttons   |
| `/about`     | About     | Bio, credentials and office info for Mr. Ernest                    |
| `/contact`   | Contact   | Contact card, enquiry form (opens WhatsApp), Google Map, socials  |
| `/faq`       | FAQ       | Accordion of common questions                                     |

Global features: sticky navbar with mobile menu, dismissible deadline banner,
floating WhatsApp button, and a footer with full contact details.

## Getting started

You need [Node.js](https://nodejs.org) 18+ installed.

```bash
# 1. Install dependencies
npm install

# 2. Start the dev server (http://localhost:5173)
npm run dev

# 3. Build for production (outputs to /dist)
npm run build

# 4. Preview the production build locally
npm run preview
```

---

## Deploying to Netlify (free)

**Option A — drag & drop**

1. Run `npm run build`.
2. Go to <https://app.netlify.com/drop> and drag the generated `dist` folder in.

**Option B — connect a Git repo (recommended)**

1. Push this project to GitHub/GitLab.
2. In Netlify, choose **Add new site → Import an existing project**.
3. Build command: `npm run build` — Publish directory: `dist`.

The included `netlify.toml` and `public/_redirects` already handle SPA routing,
so deep links like `/services` work after refresh.

## Editing content

Almost everything is data-driven so you rarely need to touch the components:

- **Contact details & WhatsApp number** → `src/data/business.ts`
  (one place: phone, emails, address, hours, and the WhatsApp number used for
  every link).
- **Services & prices** → `src/data/services.ts`
  (add/edit a service object; categories and the filter bar update automatically).
- **Brand colours & fonts** → `tailwind.config.js`.
- **Deadline banner text** → `src/components/DeadlineBanner.tsx`.
- **Ticker messages** → `src/components/Ticker.tsx`.
- **Testimonials** → the `TESTIMONIALS` array in `src/pages/Home.tsx`.
- **FAQ questions** → the `FAQS` array in `src/pages/FAQ.tsx`.

### Adding the real photo / logo

- The About page uses an "EA" placeholder circle. Drop a photo into `public/`
  (e.g. `public/ernest.jpg`) and replace the placeholder block in
  `src/pages/About.tsx` with an `<img src="/ernest.jpg" .../>`.
- The text logo lives in `src/components/Logo.tsx`. To use an image logo, place
  it in `public/` and swap the markup there.

### WhatsApp pre-fill

All order buttons open:

```
https://wa.me/2348159074358?text=Hello%20Mr.%20Ernest%2C%20I%20want%20to%20enquire%20about%20[SERVICE]
```

The number and message templates are generated by the helpers in
`src/data/business.ts` (`orderLink`, `whatsappLink`).
